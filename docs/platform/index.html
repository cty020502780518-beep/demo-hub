技术说明

          

            本页面为面试展示 Demo，当前使用前端 mock 数据展示zhimeng-ai
            工作流平台的核心链路。真实工程中，前端画布会生成 DSL JSON，Console Hub
            负责保存 DSL，Workflow Engine 负责解析 nodes 和 edges，并根据 nodeType
            调度对应 NodeExecutor。Writer Agent 生成文稿后，结果写入变量池；Reviewer
            Agent 从变量池读取文稿并输出 approved、problems、rewriteSuggestions；Workflow
            Engine 根据 approved 字段选择条件边，实现审核不通过返回 Writer
            重写、审核通过进入 Output 的图编排流程。
